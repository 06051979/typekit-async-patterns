Typekit async loading pattern: Script tag with async attribute

Adds the "async" property to the standard Typekit snippet, which makes it asynchronous in browsers that support the property. Fonts don't load at all in IE 6-8 because they don't support an onload property on the script tag.

Recommendations:

  • Place this snippet at the top of the <head> so the download starts as soon as possible

Advantages:

  • Loads Typekit fonts asynchronously (doesn't block page rendering while it loads) in Firefox 3.6+, Safari 4+, Opera 10.5+, and Chrome
  • Fewer bytes than the standard Typekit snippet

Disadvantages:

  • Typekit fonts don't load at all in IE 6-8 (these browsers don't support onload on <script>)
  • Typekit fonts load in IE9, but not asynchronously (IE9 doesn't support the async property)
  • Causes a flash of unstyled text (FOUT) in browsers that support the async attribute that can't be controlled or hidden
